# Contributing to OpenSplit

Thank you for your interest in contributing!

## How to Contribute

1. **Fork** the repository and create a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Make your changes with clear, descriptive commits.
3. Run the app and verify your changes work end-to-end.
4. **Open a Pull Request** describing what you changed and why.

## Branch Naming

| Purpose | Pattern |
|---------|---------|
| New feature | `feature/<name>` |
| Bug fix | `fix/<name>` |
| Documentation | `docs/<name>` |

## Commit Message Style

Use concise imperative messages:
- `Add login functionality`
- `Fix balance calculation for unequal splits`
- `Update group members API route`

## Code Style

- 2-space indentation
- Meaningful variable names
- Keep functions small and focused
- No unused imports or console.log in production code

## Reporting Issues

Open a GitHub Issue with:
- A clear title
- Steps to reproduce
- Expected vs actual behaviour
- Screenshots if relevant

## Code of Conduct

Please read [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before contributing.
